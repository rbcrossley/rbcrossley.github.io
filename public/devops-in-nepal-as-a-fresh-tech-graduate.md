## Introduction

For a fresh computer engineer or IT graduate, there are several available job roles in Kathmandu Valley. It could be devops, QA,networking, system admin, or software development.

The first job is very important because it is your blueprint for getting the next job. I personally have experienced that it is tough to get hired at some other jobs that does not match with your first job.

After you work for couple of years, you will not feel like starting as an intern in a different field.

Even if you do, companies will reject your resume at first sight because they think you are overqualified for the role.

Many students struggle with programming real world software. And it is normal. I, for instance can develop softwares but it takes me too much work to get little bit result. Thus I decided to invest my time in somewhere else, like devops.

# DevOps and Programming

When graduates say that they want to do DevOps, their main pain point is that they are bad at coding. Devops requires coding. So it is some bad news for you. But there is also some good news. The coding required for DevOps vs backend software development is completely different. In DevOps, you are mostly writing shell scripts, or some yamls. You just tell the computer what you want. You do not have to build the logic. Plus, scripts are write once, use forever unlike real world software which needs to be maintained. Software is used by humans directly. Hence even if you are not comfortable with software development, you can 
still be able to code in DevOps. 


Here is a specific DevOps job description from a wallet company named MOCO. This might help you understand DevOps role more than a wall of text could ever.

**Duties and Responsibilities:**

- Assist in the maintenance of our production and test environments, including network, servers, databases, and applications.
- Support the deployment of new applications and services, utilizing CI/CD pipelines.
- Manage virtualized environments using Proxmox VE and oversee backup and recovery operations with Proxmox Backup Server (PBS). This includes managing our high-performance, distributed storage cluster built with Ceph.
- Help manage our multi-site network, including configuring routers, FortiGate firewalls, and IPsec VPNs. Knowledge of Software-Defined Networking (SDN) is a plus.
- Assist with system security audits, implement WAF tools for web application protection, and manage Wazuh for security monitoring.
- Monitor system health and performance using an Observability approach, with a focus on the ELK stack for centralized logging and analysis.
- Automate repetitive tasks by writing scripts using Bash and Ansible.
- Maintain accurate network and system documentation and provide support to internal employees as needed.

**Specific Skills:**

- Good command of Ubuntu server management and proficiency in Linux environments.
- Foundational knowledge of networking principles and SDN concepts.
- Experience with FortiGate and Mikrotik devices.
- Familiarity with KVM virtualization platforms like Proxmox VE
- Familiarity with distributed storage solutions like Ceph.
- Experience with backup solutions such as Proxmox Backup Server (PBS).
- Basic Understanding of CI/CD pipelines and automation tools such as Ansible and bash scripting.
- Knowledge of SIEM and WAF tools.
- Experience with the ELK stack (Elasticsearch, Logstash, Kibana) for logging and monitoring.
- Knowledge of web servers like Apache.
- Experience with MySQL or other relational databases.


**Soft Skills:**

- Strong analytical and problem-solving skills.
- Excellent communication skills, with the ability to convey information clearly and concisely.
- Ability to work effectively under pressure.
- Flexible and willing to respond in odd hours when required.	
- Experience: At least 1 year

# Companies that hire DevOps engineers in Nepal

You should understand where DevOps engineers are required.

They are required in places where software is being maintained in-house. They are required in places where a real customer is using the software. 

I have listed lots of Nepali IT companies in the blog below:

https://berojgarengineer.com/blog/list-of-actively-hiring-it-companies-in-nepal/


From that, pick companies that 
- either have a product for users in Nepal
- or are off-shored i.e., complete software development activities are off-shored including DevOps.

Examples of such companies could be Khalti and Logpoint.

DevOps being a security critical job, true remote devops jobs are less likely. However you can do consultation, one-off deployments for clients remotely as well.

# workload
Note that labor laws are pretty loose in Nepal. On top of that, DevOps is a role that is done in shifts in the USA, and other developed countries. Since Nepalese companies are greedy, they will expect you to be 24x7 available. But they will only pay when you do the actual work, not for the time you have been in stand by.

But this is not true for developers. Specially if the software is made configurable which is the case in many big product based companies in Nepal. Support engineers will edit the configurations and keep the system up and running for the time being so that developers do not have to wake up at the middle of the night for fixing the system down issue.
